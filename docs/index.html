1. HTML: 网页的骨骼 🦴

  • HyperText Markup Language
  • 核心任务： 定义网页内容的含义和结构。
  • 不是编程语言，而是标记语言。
  • 它告诉浏览器：
    • 这是一张图片 <img>
    • 这是一个标题 <h1>
    • 这是一个按钮 <button>

HTML 实例：构建一个“素颜”卡片

为什么标签不能乱用？(语义化)

虽然用 <div> 配合样式也能做成标题的样子，但我们推荐使用正确的标签：


  • SEO (搜索引擎优化)： Google 爬虫通过 <h1> 知道这是页面的重点。
  • 无障碍访问 (A11y)： 盲人阅读器通过 <nav> 知道这是导航栏，通过 <button> 知道这是可以点击的。
代码如人： 结构清晰，不仅机器读得懂，同事也读得懂。

2. CSS: 赋予页面美感与布局 🎨

  • Cascading Style Sheets
  • 核心任务： 决定 HTML 元素看起来是什么样。
  • 它能做什么？
    • 🖌️ 美化： 颜色、字体、阴影、圆角。
    • 📐 布局： 让元素横向排列、居中、响应式调整。
    • 🎬 动画： 过渡效果、关键帧动画。

CSS 实例：给卡片“化妆”

通过选择器选中 HTML 标签，并应用属性。 


/* 选中卡片容器 */
.profile-card {
  background: white;
  border-radius: 12px;       /* 圆角 */
  box-shadow: 0 4px 10px rgba(0,0,0,0.1); /* 阴影让它浮起来 */
  text-align: center;        /* 让文字居中 */
  width: 300px;
}

/* 选中里面的按钮 */
.profile-card button {
  background-color: #3b82f6; /* 蓝色背景 */
  color: white;
  border: none;
  padding: 10px 24px;
  cursor: pointer;           /* 鼠标放上去变小手 */
}

布局神器：Flexbox & Grid

以前我们要把两个东西并排很难，现在很简单：


.container {
  display: flex;           /* 开启弹性布局模式 */
  justify-content: center; /* 主轴居中（水平） */
  align-items: center;     /* 交叉轴居中（垂直） */
  gap: 20px;               /* 元素之间的间距 */
}

CSS 让网页从“文本文档”变成了“精美的应用程序界面”。

3. JavaScript: 注入灵魂与交互 ⚡

  • 网页的编程语言（拥有逻辑、循环、变量、函数）。
  • 核心任务： 响应用户操作，动态修改页面。
  • 没有 JS 的页面是静态的： 点击按钮没有任何反应，数据无法更新。
  • 有了 JS：
    • 🖱️ 响应点击、滚动、键盘输入。
    • 📡 发送网络请求 (API)。
    • 🔄 不刷新页面的情况下更新内容。

JS 实例：让按钮“活”过来

我们需要监听用户的点击，并执行逻辑。


// 1. 获取元素：在文档中找到那个按钮
const btn = document.querySelector('button');
const title = document.querySelector('h2');

// 2. 监听事件：当用户点击时...
btn.addEventListener('click', () => {
  
  // 3. 执行逻辑：修改 HTML 内容和 CSS 样式
  btn.textContent = '已发送请求...';
  btn.style.backgroundColor = '#ccc'; // 按钮变灰
  
  // 模拟简单的交互反馈
  alert(`已向 ${title.textContent} 发送好友申请！`);
});

幕后机制：DOM (文档对象模型)

JavaScript 并不直接操作 HTML 文本文件，它操作的是内存中的 DOM 树。

流程图解

用户点击 ➡️ 浏览器触发 JS 事件 ➡️ JS 修改 DOM 节点 ➡️ 浏览器重新绘制页面

  • HTML 是建筑图纸。
  • DOM 是盖好的房子。
  • JS 是可以随意装修、改建的施工队。

Vue.js 3 🟢

渐进式 JavaScript 框架 (The Progressive Framework)

  • 🚀 易学易用： 建立在标准 HTML/CSS/JS 之上。
  • ⚡ 高性能： 虚拟 DOM + 编译时优化。
  • 🧩 组件化： 像搭积木一样构建用户界面。

“不只是写代码，更是在描述你的想法。”

Vue 的核心哲学：声明式 vs 命令式

原生 JS (命令式) 就像做菜，必须一步步教浏览器怎么做： 


// 命令式 (Imperative)
const div = document.querySelector('#app');
div.innerText = 'Hello'; // 手动找到元素，手动修改

Vue (声明式) 就像点菜，你告诉它结果，它自己去处理过程： 





{{ message }}
核心魔法：数据驱动视图 (Data-Driven View)
数据变了 ➡️ 界面自动更新。再也不用写 document.querySelector 了！

SFC: 单文件组件 (.vue)

Vue 将一个组件的代码封装在一个 .vue 文件中，结构清晰：

App.vue
  • <script setup>: 编写业务逻辑 (Composition API)。
  • <template>: 定义 HTML 模板。
  • <style scoped>: 定义样式 (scoped 代表只在当前组件生效，不污染全局)。

Vue 3 的心脏：响应式系统 ✨

在 JS 中，变量变了界面是不会动的。Vue 3 使用 ref() 来包裹变量，让它变成“响应式”的。 


import { ref } from 'vue';

// ref 让普通数字变成了“可被监听”的数据
const count = ref(0); 

function increment() {
  // 在 JS 中修改值需要加 .value
  count.value++; 
  // 🪄 此时，Vue 自动检测到变化，并更新页面上所有用到 count 的地方
}

为什么是 Vue 3？

  • Vue 2 使用 Object.defineProperty (有局限性)。
  • Vue 3 使用 ES6 Proxy，性能更强，能监听更多类型的数据变化。

模板语法：指令 (Directives)

Vue 在 HTML 中提供了一套特殊的属性（以 v- 开头）来处理常见的 DOM 逻辑：

v-if / v-else 条件渲染 <div v-if="isLogin">欢迎回来</div>
v-for 列表循环 <li v-for="item in list">{{ item.name }}</li>
v-on (@) 事件监听 <button @click="submit">提交</button>
v-model 双向绑定 (表单神器) <input v-model="username">
(输入框变->数据变；数据变->输入框变)

Vue 3 的杀手锏：Composables

在大型项目中，我们希望复用逻辑（比如“获取鼠标位置”）。


// mouse.js (自定义 Hook)
import { ref, onMounted, onUnmounted } from 'vue'

export function useMouse() {
  const x = ref(0)
  const y = ref(0)

  function update(event) {
    x.value = event.pageX
    y.value = event.pageY
  }

  onMounted(() => window.addEventListener('mousemove', update))
  onUnmounted(() => window.removeEventListener('mousemove', update))

  return { x, y }
}

这是 Vue 3 比 Vue 2 更有利于大型项目维护的关键。

从原生 JS 到 Vue 3 的进化

  • 操作对象： DOM 元素 ➡️ 业务数据 (State)
  • 关注点： "怎么修改这个 div" ➡️ "这个 div 应该显示什么状态"
  • 代码组织： 散落在各处的函数 ➡️ 高内聚的组件 (SFC)
  • 复用性： 复制粘贴代码 ➡️ 组合式函数 (Composables)
